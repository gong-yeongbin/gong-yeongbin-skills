# create-issue

당신의 임무는 사용자가 준 **한 줄짜리 작업 설명**을, Claude Code의 **agent teams** 기능으로 팀을 꾸려 토론시킨 뒤, 실행 가능한 **이슈 초안**으로 확장하는 것이다. 당신(현재 세션)은 팀의 **리더**다. 세부 작업을 직접 다 파악하려 들지 말고, 팀원 2명이 서로 다른 관점에서 파악하고 **토론으로 보완**하게 만든 다음, 그 결과를 정리하는 데 집중한다.

## 전제조건

이 스킬은 Claude Code 실험 기능 **agent teams**가 활성화돼 있어야 동작한다. 팀원 spawn이나 팀원 간 메시지가 되지 않으면, 기능이 꺼져 있을 가능성이 높다. 그때는 작업을 강행하지 말고, README의 "create-issue — agent teams 전제조건" 안내(`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` 활성화)를 사용자에게 알린 뒤 멈춘다.

## 실행 흐름

### 0. 작업 설명 수집

- 사용자가 이미 작업 설명을 줬으면 그대로 쓴다.
- 안 줬으면 **한 줄로** 묻는다: "어떤 작업을 이슈로 정리할까요? 한두 문장으로 알려주세요." 그 이상 캐묻지 않는다. 세부는 팀이 파악할 몫이다.

### 1. 팀 구성 (리더 = 당신)

팀원 **2명**을 자연어로 spawn한다. 두 팀원은 **서로 다른 관점**을 맡아야 토론이 형식적으로 흐르지 않는다.

- **팀원 A — 범위·구현 관점.** "무엇을, 어디서, 어떻게 바꿔야 하는가"에 집중한다. 구현 단계, 손대야 할 파일·모듈·인터페이스, 작업을 쪼갠 하위 항목을 도출한다.
- **팀원 B — 리스크·검증 관점.** "무엇이 잘못될 수 있고, 무엇으로 완료를 판정하는가"에 집중한다. 엣지케이스, 의존성, 실패 시나리오, 완료(수용) 기준을 도출한다.

각 팀원에게 동일한 작업 설명을 전달하되, 위 관점과 "다른 팀원과 토론해 결론을 보완하라"는 지시를 함께 준다.

### 2. 독립 분석

두 팀원이 **서로의 결과를 보기 전에** 각자 작업 설명만으로 세부 작업 목록을 파악한다. 코드나 파일을 읽어 확인할 수 있는 건 추측하지 말고 조사하게 한다.

### 3. 토론

팀원들이 서로에게 메시지를 보내 각자의 분석을 공유하고, **실제로 이견을 드러내며** 도전·보완한다:

- 상대 관점에서 빠진 항목을 지적하고, 근거 없는 항목엔 근거를 요구한다.
- A의 구현 항목이 B가 짚은 리스크·완료 기준을 충족하는지 교차 검증한다.
- 리더인 당신은 토론을 조정한다. 한쪽으로 쏠리거나 겉핥기로 합의하면 다시 파고들게 한다.
- **수렴 조건:** 두 관점이 실질적으로 합의에 도달하거나, 최대 2~3라운드가 지나면 종료한다. 최소 한 번은 실질적 이견 또는 보완이 오가야 한다.

### 4. 취합 (리더)

토론 결과를 **이슈 형태**로 정리한다. 두 관점이 합쳐진 항목만 최종에 넣는다.

- **제목** — 작업을 한 줄로 요약.
- **배경·목적** — 왜 이 작업이 필요한가, 무엇을 이루려는가.
- **세부 작업 항목** — 체크리스트(`- [ ]`)로. 구현 단위로 쪼갠 실행 항목.
- **고려사항·리스크** — 엣지케이스, 의존성, 주의할 점.
- **완료 기준** — 무엇이 충족되면 이 작업이 끝난 것으로 보는가(수용 기준).

### 5. 출력

정리한 이슈 초안을 **채팅에 출력**한다. 지금은 파일이나 GitHub 이슈를 만들지 않는다. 다만 위 5개 섹션 구조를 **일관되게** 유지해, 추후 이 내용을 이슈 등록 등에 그대로 재활용할 수 있게 한다.

## 품질 규칙

- 토론이 요식행위가 되지 않게 한다. "둘 다 동의합니다"로 즉시 끝나면 리더가 다시 파고들게 한다.
- 최종 이슈에는 **두 관점의 검증을 통과한 항목**만 넣는다. 한쪽만 주장한 미검증 항목은 "고려사항"으로 내리거나 뺀다.
- 팀원이 코드·파일로 확인 가능한 사실은 추측하지 말고 조사하게 한다.

## 이 스킬이 아닌 것

- **곧바로 코드를 구현하는 스킬이 아니다.** 여기서의 산출물은 이슈 초안까지다.
- **이슈를 실제로 등록하는 도구가 아니다.** 현재는 채팅에 정리 출력까지만 한다.
